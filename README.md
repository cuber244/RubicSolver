## 基本の使い方

もっとも基本的な流れは次のとおりです。

1. **「WCAスクランブル生成」** を押す
   → ランダムなスクランブルが作られ、3D キューブがその手順で崩れます
2. 解きたい方法のタブ（**CFOP / Roux / 最短**）を選ぶ
3. **「解法を生成」** を押す
   → 右側にフェーズごとの手順が表示されます
4. 再生バーの **▶** で自動再生、**▶▶** で1手ずつ進めて確認できます

すでに手元に崩れたキューブがある場合は、[展開図エディタ](#展開図エディタ自分でキューブ状態を入力)
でその状態を入力してから「解法を生成」を押します。

## 3D ビューの操作

- **ドラッグ（マウス左ボタン / タッチ）**: キューブを回して好きな向きから見られます
  - 横ドラッグ … 上下方向を軸に水平回転
  - 縦ドラッグ … 画面の横方向を軸に回転（上下 360° 制限なし）
  - カメラは固定でキューブ自体が回るため、どの向きでもドラッグ方向と回転が一致します
- **マウスホイール**: ズームイン / アウト
- **「視点リセット」ボタン**（3D 枠の右上）: 視点を初期状態に戻します
- タブレット・スマホでは、キューブを回している間はページがスクロールしないようになっています

## 解法モード（CFOP / Roux / 最短）

| モード | 手法 | 平均手数 |
|---|---|---|
| **CFOP** | Cross → F2L → OLL → PLL | 約60手 |
| **Roux** | FB → SB → CMLL → LSE | 約55手 |
| **最短** | Kociemba two-phase | 約20手 |

CFOP / Roux は人間の実戦と同じく **白面を下・緑面を正面** にして解きます。
解法の先頭に持ち替え `z2`（「持ち替え」フェーズ、手数には数えません）が入り、
クロス／第1ブロックは白面側、OLL/PLL/CMLL は黄色面側で行われます。

**CFOP**（初中級者に人気の方法）

1. **Cross**: 白いクロス（十字）を作る
2. **F2L**: コーナーとエッジをペアにして4スロットを埋める（1組ずつ表示）
3. **OLL**: 黄色面の向きをそろえる（57 手順）
4. **PLL**: 最後に位置を入れ替えて完成（21 手順）

**Roux**（少ない手数で解ける方法）— 各段階をさらに細かく提示します

1. **FB（左ブロック）**: DL エッジ → やさしい方のペア → 残りのペア
2. **SB（右ブロック）**: DR エッジ → やさしい方のペア → 残りのペア
3. **CMLL**: 上のコーナー4つを一気にそろえる（42 手順）
4. **LSE（最後の6エッジ）**: 4a エッジの向き → 4b 左右エッジ → 4c 仕上げ

**最短**: 世界的に使われる two-phase アルゴリズムで、20手前後の短い手順を出します
（人間向けの手筋ではなく、手数を優先した機械的な手順です）。

## 解法の読み方（記法とフェーズ）

解法は **フェーズごとのラベル**（例: `Cross (白面)`、`F2L — #1 FR`、`OLL (黄色面)`）と、
その手順の **ムーブチップ** で表示されます。手順の総手数は上部に「解法: NN 手」と出ます。

**ムーブ記法（Singmaster 記法 / WCA 公式）**

| 記号 | 意味 |
|---|---|
| `U D L R F B` | 各面（上・下・左・右・前・後）を時計回りに90° |
| `'`（例 `R'`） | 反時計回りに90° |
| `2`（例 `U2`） | 180° |
| `M E S` | 中央の列（スライス）を回す |
| `x y z` | キューブ全体の持ち替え（回転）。手数には数えません |

- **ムーブチップをクリック** すると、その手の直後まで一気に進みます
- 再生中は現在の手が青くハイライトされ、済んだ手は薄く表示されます

## スクランブルの入力・生成

- **「WCAスクランブル生成」**: WCA 現行規則の**ランダムステート方式**で正当なスクランブルを生成し、
  3D キューブに適用します
- **入力欄に手入力**: `R U R' U' F2 D ...` のように記法で入力し、**「適用」** を押すと
  完成状態からその手順を回した状態になります
- **「リセット」**: キューブを完成状態に戻します

## カメラで状態を読み取る

手元の実物キューブを、スマホの背面カメラで撮って登録できます。
スクランブル欄の下の **「📷 カメラで読み取る」** を押すと、カメラ画面になります。

1. 画面中央の **3×3 のガイド線** に、キューブの1面をぴったり合わせます
2. 画面下の案内（「好きな面を…」「下の面を手前に倒して…」など）に従い、
   **「撮影」** を押して6面を順に撮ります
   （最初の面 → 下・下・下 → 右 → 奥、と持ち替えるだけ）
3. 画面の隅に **展開図** が常に表示され、撮影した面が色で埋まっていきます
4. 6面撮り終えると自動でキューブの状態を判定し、問題なければ
   **「この状態で決定」** で 3D に読み込まれます

**うまく読み取れないとき**

- 各マスの色は必ず **白・黄・緑・青・赤・橙の6色** のいずれかに丸められます。
  展開図の色が実物と違う面があれば、**その面を展開図でタップ** すると撮り直せます
- 明るい場所で、影や光の反射が入らないように撮ると精度が上がります

> ⚠️ カメラ機能は、ブラウザの仕様上 **ダブルクリック（`file://`）では使えません。**
> `https://` で公開したページ（GitHub Pages など）をスマホで開いてお使いください。

## 展開図エディタ（自分でキューブ状態を入力）

画面右下の「展開図エディタ」を開くと、キューブの展開図（54 マス）が表示されます。

1. 上のパレットから **色を選ぶ**
2. 展開図の **マスをクリック** して色を塗る（中央のマスは固定で変更できません）
3. **「キューブへ反映」** を押すと、その状態が 3D に読み込まれます

塗った状態が物理的にありえない配色（色の枚数が合わない、パリティ不正など）の場合は、
エラーメッセージで理由が表示されます。手元の崩れたキューブを解きたいときに使います。

## スマートキューブ（GAN356i など）連携

Bluetooth 対応の GAN スマートキューブと接続すると、実機を回すと 3D 表示が追従し、
解法を **1手ずつリアルタイムでアシスト** できます
（GAN356 i / i3 / i Carry, GAN12 ui 系などに対応）。

> ⚠️ **重要**: この機能はブラウザの仕様上、**ダブルクリック（`file://`）では使えない場合があります。**
> スマートキューブ機能を使うときは、`https://` で公開したページ（GitHub Pages など）を
> **PC 版 Chrome / Edge** で開いてください。解法生成・3D 表示・スクランブルなど、
> その他の機能はこのスタンドアロン版でそのまま使えます。

**接続と使い方**

1. 「スマートキューブ」パネルの **「接続」** を押し、デバイスを選択
   （MAC アドレスは自動取得を試み、失敗した場合のみ手入力。次回のために保存されます）
2. 接続すると、実機を回すたびに 3D 表示が追従し、電池残量も表示されます
3. 実機を崩して **「解法を生成」** を押すと、アシストが始まります

**アシスト中の案内**

- まず「白面を下・緑面を正面」に **持ち替える** よう案内し、以降の手順もその持ち方の
  面記号で表示します（3D 表示も同じ向きに回転します）
- 次に回すべき面が **3D 上でループアニメーション** で示されます
- 手順どおり回すと自動で次の手へ進みます
- **間違えたとき** は、元に戻すための**最短の手順**が差し込み表示されます
  （例: R を誤って3回回した場合、R' を3回ではなく「R あと1回」で復帰）
- 手順に従わず自力でクロスやブロックなどの**区切りまで揃えた場合**も検知し、
  その状態から**自動で解き直して**続きを案内します

**その他のボタン**

- **「状態を再読込」**: 実機の状態を取り直して 3D と同期（ずれの補正）
- **「実機を完成状態として登録」**: 実機を物理的に完成させてから内部状態をリセット
